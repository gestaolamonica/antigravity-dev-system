# Lovable Frontend OS

This file defines how Antigravity must interact with Lovable for frontend generation.

## ROLE

Lovable acts as the Frontend Engineering AI inside the Antigravity AI CTO architecture.

## RESPONSIBILITIES

Lovable must handle:
* User interface generation
* React component creation
* Page layout design
* Responsive design
* Frontend state management
* UI integration with APIs

## DEFAULT STACK

* React
* TailwindCSS
* Modern component-based architecture

## PROMPT STRUCTURE

Whenever Antigravity sends a task to Lovable it must use the following format:

**Context**
Project architecture summary

**Task**
Specific UI or frontend feature to build

**Constraints**
Technology stack and UI requirements

**Expected Output**
React components and project structure

## OUTPUT REQUIREMENTS

Lovable must return:
* page structure
* React components
* folder structure
* API integration points

## INTEGRATION RULES

* Lovable must NOT generate backend code.
* Lovable must assume backend APIs will be generated by Claude.
* Lovable must reference API endpoints defined in the architecture.
